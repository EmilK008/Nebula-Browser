        Bookmarks

        Privacy

        Saved passwords are stored only on this device. Use View → Saved passwords or the button below. Nebula can also suggest a vault row when you are signed in via cookies but have no entry yet (password stays empty until you edit it)—see View → Add this site to vault (session login). Removing a session row also clears that site’s cookies and data on this device (you are signed out there).

        Local Nebula account

        Optional master password on this device. Until you unlock, saved passwords stay hidden and cannot be copied or edited (you can still add new entries or remove rows). Unlock lasts about 15 minutes, then locks again.

        About

        Nebula Version —

        DRM (Crunchyroll, Netflix, Spotify): A 400 from Google’s Widevine endpoint usually means unsigned or dev-only VMP. Ship builds need Castlabs EVS (production VMP signing) on the win-unpacked folder (the directory that contains Nebula.exe), then ship that signed tree. On Windows, if you Authenticode-sign the exe, run EVS after that step; any later edit or re-sign of Nebula.exe invalidates VMP. The build runs EVS sign-pkg in an afterPack hook (before the NSIS and Portable artifacts are produced) so those installers match win-unpacked; signing only after npm run dist would leave them unsigned. After packaging, run npm run verify:vmp to confirm the signature. A 403 on a site’s own license URL is often a session, region, or app-check issue (not fixed by EVS alone).

        Bump version in package.json when you ship. In renderer/changelog.json, use optional arrays added, improvements, and fixes (or a plain items list for older-style notes).

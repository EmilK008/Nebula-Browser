        Web search (optional)

        When enabled, the assistant can call a web_search tool; queries are sent to Brave Search (get a key from Brave Search API). Stored in the vault like other keys.

        Page text fetch (optional)

        When enabled, the assistant can call fetch_page_text with an https or http URL. Nebula downloads the page in the main process (no visible tab), strips HTML to plain text, and returns a capped excerpt to the model — not your logged-in session or cookies. Private IPs and file: URLs are blocked.

        Tab agent (Tier C, optional)

        When enabled, the assistant can call open_browser_tab and read_browser_tab_text on real Nebula tabs — your profile cookies and JavaScript apply (unlike Tier A page fetch). Private IPs and non-http(s) URLs are blocked in the shell. Limits apply per assistant reply.

        Extra model IDs (one per line)

        These appear in the assistant’s model list next to built-in suggestions. Provider IDs change over time — add the exact id from your provider’s docs.

        Search

        Non-URL text in the address bar uses this engine. AI search opens the assistant and answers using web search (needs your AI API key and a Brave Search API key in AI assistant settings below). AI search uses the same address-bar suggestions as other engines (past searches, bookmarks, history, and live completions).

        Bookmarks

        Profiles & private tabs

        Profiles use separate cookies, saved sessions, bookmarks, and history (each profile has its own bar, address suggestions, and local storage keys). Switch profile below, click Save, and Nebula will restart automatically. Default migrates your previous global bookmarks and history the first time they load. Private tabs use a temporary session (Tab → New private tab or Ctrl+Shift+N); they are not restored after quit.

        Network & proxy

        This is not a built-in VPN. If you use a normal VPN app (WireGuard, OpenVPN, commercial clients), connect it at the OS level — Nebula’s tabs already follow the system tunnel. Use the options below when your VPN exposes a local SOCKS or HTTP proxy, you use a corporate PAC/proxy, or you want Nebula to follow the OS proxy configuration.

        Fixed proxy also applies to the main process (optional AI page fetch, translation requests, update checks) via the same rules — so egress matches tabs. Avoid putting passwords in the rules string; prefer SOCKS without auth or a local proxy you control.

        Privacy

        Saved passwords are stored only on this device. Use View → Saved passwords or the button below. Nebula can also suggest a vault row when you are signed in via cookies but have no entry yet (password stays empty until you edit it)—see View → Add this site to vault (session login). Removing a session row also clears that site’s cookies and data on this device (you are signed out there).

        When enabled, one unlock covers AI keys, translation keys, and viewing saved passwords for the whole session. When off, unlock lasts about 15 minutes, then locks again. You can still lock manually below.

        Local Nebula account

        Optional master password on this device. Until you unlock, saved passwords stay hidden and cannot be copied or edited (you can still add new entries or remove rows).

        Site permission overrides

        Sites where you changed camera, microphone, location, notifications, screen capture, or third-party cookies from the app default. Clearing a row removes only that site’s custom rules (defaults in the Site permissions panel are unchanged).

        Nebula Version —

        DRM (Crunchyroll, Netflix, Spotify): A 400 from Google’s Widevine endpoint usually means unsigned or dev-only VMP. Ship builds need Castlabs EVS (production VMP signing) on the win-unpacked folder (the directory that contains Nebula.exe), then ship that signed tree. On Windows, if you Authenticode-sign the exe, run EVS after that step; any later edit or re-sign of Nebula.exe invalidates VMP. The build runs EVS sign-pkg in an afterPack hook (before the NSIS and Portable artifacts are produced) so those installers match win-unpacked; signing only after npm run dist would leave them unsigned. After packaging, run npm run verify:vmp to confirm the signature. A 403 on a site’s own license URL is often a session, region, or app-check issue (not fixed by EVS alone).

        Bump version in package.json when you ship. In renderer/changelog.json, use optional arrays added, improvements, and fixes (or a plain items list for older-style notes).
